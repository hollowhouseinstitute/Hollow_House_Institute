### 1. Start with an Issue
Before making significant changes:
- Search existing issues to avoid duplication
- Open a new issue to discuss your proposed changes
- Wait for feedback from maintainers

### 2. Fork and Clone
```bash
# Fork the repository on GitHub
git clone https://github.com/YOUR-USERNAME/Hollow_House_Institute.git
cd Hollow_House_Institute
git remote add upstream https://github.com/hollowhouseinstitute/Hollow_House_Institute.git
```

### 3. Create a Branch
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

### 4. Make Your Changes
- Keep changes focused and minimal
- Follow existing code style and conventions
- Update documentation as needed
- Add or update tests if applicable

### 5. Commit Your Changes
```bash
git add .
git commit -m "Brief description of your changes"
```

Use clear, descriptive commit messages:
- âœ… "Add researcher access application template"
- âœ… "Fix broken link in ethics documentation"
- âŒ "Update stuff" or "Fixed things"

### 6. Push and Submit a Pull Request
```bash
git push origin feature/your-feature-name
```

Then open a pull request on GitHub with:
- Clear title and description
- Reference to related issues
- Explanation of changes made
- Any testing performed
